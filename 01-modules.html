Modules XX KB

Basic modules, completly optional

On top of the most basic features, Chaos Fundamentum provides completly optional modules that will ease the fast developing and prototyping of homepages and webapps. And if you don't need a module for productive use: Just don't compile it.

Grid XX KB

The simple grid system allows you to align cells inside a single row (no fancy multiline grid is provided at the moment). It uses the modern and futureproof flexbox-model. As such, there's no real fallback for old browsers possible (however they will render the content stacked vertically).

Nesting columns and auto-computing is easy.

.cell
.cell.grid
.cell
.cell
<div class="grid">
  <div class="cell"></div>
  <div class="cell"></div>
</div>

Different sizes are provided:

.cell-1of2
.cell-1of2
.cell-1of3
.cell-2of3
<div class="grid demo-grid">
  <div class="cell-1of3"><code>.cell-1of3</code></div>
  <div class="cell-2of3"><code>.cell-2of3</code></div>
</div>

It's even responsive! You can specify a breakpoint where it collapses to default block-level elements. Go ahead, try it out!

.cell-1of3
.cell-1of3
.cell-1of6
<div class="grid-md"></div>

Buttons XX KB

To style a button, just give it the btn class. This might seem redundant, but it prevents disturbance with external widgets and such, that are using their own buttons. Also you can apply this classes to inputs (submits and resets) or anchors.

<button class="btn">I'm a simple button.<button>

To apply a different design, just add one of the additional classes. But don't use them alone!

<button class="btn btn-success"></button>

A button can be disabled by applying the corresponding attribute.

<button class="btn btn-primary" disabled>I'm a disabled button.</button>

The buttons come in four (fully customizable) different (fully customizable) sizes:




 
<button class="btn btn-lg"></button>

Maybe you want to arrange them in block level? Here you go:

<button class="btn btn-block"></button>

There's also an active-state triggered by a class:

<button class="btn active"></button>

Lastly, an example for styling inputs or anchors. Please use the button element as often as you can to have a consistent and proper look.

a.btn 
<a href="#" class="btn">a.btn</a>
<button class="btn">button.btn</button>
<input type="submit" class="btn" value="input[type="submit"].btn">

Button groups

You can group buttons together by wrapping them in a button group.

<div class="btn-group">
  <button class="btn"></button>
  <button class="btn active"></button>
</div>

Groups can also be grouped: In toolbars.

<div class="btn-toolbar" role="toolbar">
  <div class="btn-group"></div>
</div>

This also works vertically:

<div class="btn-group-vertical">
  <button class="btn"></button>
</div>

Different sizings apply to all child buttons:

<div class="btn-group btn-group-lg"></div>

You can force a group to take up all available space.

<div class="btn-group btn-group-justified"></div>

Form elements XX KB

The framework provides standardised input elements. The most basic form of a form is the inline version:

<form action="" role="form" class="form-inline">
  <label for="demoInlineEmail" class="visually-hidden">Email address</label>
  <input type="email" id="demoInlineEmail" placeholder="Enter email">
  <label for="demoInlinePassword" class="visually-hidden">Password</label>
  <input type="password" id="demoInlinePassword" placeholder="Password"</label>
  <div class="checkbox"><input type="checkbox" id="demoInlineRemember"><label for="demoInlineRemember">Remember me</label></div>
  <button type="submit" class="btn btn-primary">Login</button>
</form>

Please do always provide labels for your inputs. You can hide them for visual devices by applying the visually-hidden class to them.

Without applying a class but by wrapping the form elements in a .form-group container, you will archieve a stacked formular. You can even use fieldsets and apply colors to the inputs. The form-control class will expand an input as wide as possible.
