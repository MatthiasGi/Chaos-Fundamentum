Normalization

Chaos Fundamentum is built on top of normalize.scss which is basically a port of normalize.css to SASS which makes it more customizable without the need of changing the intern code. Thus the framework does not provide a complete reset, but is built around a normalization for every browser. That means that you won't get pixel-perfect applications on each browser, but who need's that anyway? Is it even really possible?

What does normalize.(s)css do except for fixing some browser bugs? It even styles a few elements, so that there is no need to really create styles for each element there is. Also it provides a stylesheet for basic print-styles. This site won't go deeper into normalization, if you really want to know, what is set by this style sheet, check out it's repository (mentioned above).

Typography

The line height and font size are completly responsive. Try it for yourself and resize the browser and / or view the page on your mobile device. As such, most sizes are defined in percents, em or rem. The most common elements for text semantics and structuring will be showcased in this section.
