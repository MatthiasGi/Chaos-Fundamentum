Normalization

Chaos Fundamentum is built on top of normalize.scss which is basically a port of normalize.css to SASS which makes it more customizable without the need of changing the intern code. Thus the framework does not provide a complete reset, but is built around a normalization for every browser. That means that you won't get pixel-perfect applications on each browser, but who need's that anyway? Is it even really possible?

What does normalize.(s)css do except for fixing some browser bugs? It even styles a few elements, so that there is no need to really create styles for each element there is. Also it provides a stylesheet for basic print-styles. This site won't go deeper into normalization, if you really want to know, what is set by this style sheet, check out it's repository (mentioned above).

Typography

The line height and font size are completly responsive. Try it for yourself and resize the browser and / or view the page on your mobile device. As such, most sizes are defined in percents, em or rem. The most common elements for text semantics and structuring will be showcased in this section.

Utilities

Fundamentum provides you with different utility classes to eliminate the need of big code changes. They will be presented below.

Color & alignment

On compiling Chaos Fundamentum you have the choice of selecting a color scheme. The default one provides enough colors for nearly all situations. One way of actually using this scheme is by coloring text. You can also align text with various classes as in this example:

<p class="text-warning text-justify">Justified warning text</p>
<p class="text-danger">Danger text</p>
<p class="text-muted">Muted text</p>

To float items, use the pull-left and pull-right classes. There is also a clearfix class available for the parent container.

If you want a proper container for your content, wrap it in a div with the container class. This will center the container and also calculate a size according to the device width. There's also a card class, which has a slight border and additional padding. As the background can be selected independently by the body background, it can be used to create a more native feel for mobile applications.

To display block type elements inline, just give them the embedded class, which will overwrite the display with inline-block.

To completly hide an element, apply the hidden attribute or class. To only hide it but make it accessible for screen-readers and similar, apply the visually-hidden class to it.

There are also print classes. Similiar to the example above, the corresponding classes are hidden-print and visible-print.
